`open-multi-agent` 是面向 Node.js 后端的 TypeScript 原生编排框架。它让不同专长的智能体组成一支团队：面对非简单目标，`runTeam()` 可以把结果要求拆成任务 DAG，并行执行相互独立的工作，再汇总各项结果。

:::tip[从目标出发，而不是先画流程图]
描述团队要实现什么，OMA 可以在运行时决定如何分工。如果任务很简单，或者任务图已经确定，也可以选择更轻量或更明确的运行方式。
:::

## OMA 适合你吗？

<CardGrid>
  <Card title="适合使用 OMA" icon="approve-check">
    - 一个目标需要多种专业角色协作。
    - 部分工作可以并行执行。
    - 执行步骤会随输入变化。
    - 希望把编排直接嵌入现有 Node.js 后端。
  </Card>
  <Card title="保持简单会更合适" icon="laptop">
    - 调用一次模型就能解决任务。
    - 每一步和每次跳转都必须固定。
    - 只需要模型 SDK 或聊天界面。
    - 不希望承担额外的规划调用成本。
  </Card>
</CardGrid>

## 一次运行分成四步

1. **目标**——应用提供期望结果和一组可用智能体。
2. **规划**——面对非简单的 `runTeam()` 目标，协调器生成任务、依赖关系和分派；简单目标可能直接交给一个智能体。
3. **执行**——调度器在依赖完成后解锁任务，并行运行互不依赖的工作。
4. **结果**——`runTeam()` 汇总各任务输出；如果为智能体配置 `outputSchema`，单个智能体也可以返回经过 schema 校验的结构化数据。

[架构页](/zh/architecture/)展示了这条流程背后的运行时组件。

## 四个核心概念

<CardGrid>
  <Card title="智能体（Agent）" icon="laptop">
    一个明确的角色，包含模型、指令和显式授权的工具。
  </Card>
  <Card title="团队（Team）" icon="random">
    一次目标可调用的智能体集合，可以选择共享内存。
  </Card>
  <Card title="任务（Task）" icon="setting">
    带负责人和可选依赖关系的一项工作。
  </Card>
  <Card title="编排器（Orchestrator）" icon="approve-check">
    创建团队、调度工作并返回运行结果的 API。
  </Card>
</CardGrid>

## 选择起步方式

- 一个智能体处理一项明确任务：使用 `runAgent()`。
- 知道目标，但不想维护任务图：从 `runTeam()` 开始。
- 任务图和分派必须明确：使用 `runTasks()`。
